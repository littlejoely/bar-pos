# 安全策略

## 支持范围

安全修复以默认分支的最新版本为准。历史提交、个人分支和未经修改的第三方依赖不单独维护。

## 报告漏洞

请优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告功能。若该入口尚未启用，请先通过仓库所有者的公开主页请求建立私密联系，不要在公开 Issue、讨论区或 Pull Request 中披露以下内容：

- 可直接复现的攻击步骤或利用代码；
- 真实账号、密码、短密码、Cookie、令牌、数据库或服务器凭据；
- 未脱敏的订单、用户、日志或服务器数据；
- 尚未修复的越权、支付金额篡改、会话劫持或远程执行细节。

报告应包含受影响版本、影响范围、最小复现步骤、预期与实际结果，以及可行的缓解建议。维护者确认问题并准备修复前，请暂缓公开披露。

## 部署责任

本项目默认配置面向本地开发。生产部署至少应使用 HTTPS，设置 `POS_COOKIE_SECURE=true`，限制 `POS_CORS_ORIGINS`，妥善保护 `.env` 与认证数据库，并建立备份恢复、日志监控和依赖更新流程。不得将示例凭据、服务器密码或真实密钥提交到仓库。
